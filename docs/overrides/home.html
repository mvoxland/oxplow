// oxplow

The cockpit for agent-assisted development.

One human. Several agents. One codebase. Oxplow gives you parallel streams, durable work queues, a project wiki with backlinks, and a Local History timeline — wired together as a web of pages so your agents do the typing while you stay focused on the task.

// what oxplow gives you

Streams + worktrees

Each stream owns a branch and an isolated checkout. Agents on different streams cannot clobber each other.

Threads inside streams

One writer per stream that ships changes; optional read-only research threads with their own agents and tabs.

Durable work queue

Tasks with a real lifecycle — ready, in_progress, blocked, done — grouped by epic, enforced by hooks.

Project wiki

Markdown notes with [[wikilinks]] to other notes, files, and commits — and cross-kind backlinks everywhere.

Local History

Every effort snapshots the files it touched. Browse snapshots by recent activity or by the git commit they pin to, drill into per-snapshot detail pages, restore a single file without disturbing the rest.

Change Analysis

A diff-aware dashboard that ranks files by interestingness (churn × complexity × tests-missing × duplication), with drilldown by extension / directory / status and per-function before/after metrics.

Web-style UI

Pages, tabs per thread, browser back/forward, kebabs instead of modals, drag-to-add-context onto the agent. The whole project is one navigable graph.
